Network Architecture Map
=======================
Last updated: 2026-06-01

All services use per-project internal networks (172.20-34.0.0/16) plus
the shared traefik edge proxy network (172.20.0.0/24).

Services using network_mode: service:<parent> share the parent's network
stack and are reached via the parent's hostname/IP.


SHARED TRAEFIK NETWORK (172.20.0.0/24)
--------------------------------------
Managed by manage-compose.sh _init_traefik_network().
Static IPs for runsc-systrap containers caused AddressAlreadyInUse errors;
they now use auto-assigned (DHCP) IPs. Only runc containers keep static IPs.

  STATIC:
  172.20.0.1   gateway (Docker bridge)
  172.20.0.2   wg1        (wg1_qbt,     runtime: runc)
  172.20.0.3   wg2        (wg2_usenet,  runtime: runc)
  172.20.0.4   wg3        (wg3_general, runtime: runc)

  AUTO (no static IP — assigned by Docker IPAM):
  syncthing1   (syncthing,  runtime: runsc-systrap)
  syncthing2   (syncthing,  runtime: runsc-systrap)
  pypowerwall  (powerwall,  runtime: runsc-systrap)
  grafana      (powerwall,  runtime: runsc-systrap)
  vault        (vault,      runtime: runsc-systrap)
  flexo        (flexo,      runtime: runsc-systrap)
  mpd          (music,      runtime: runc — app-runc, auto IP)

SERVICES WITHOUT ROUTABLE STATIC IPs
------------------------------------
These extend 'app' profile and use default bridge or no network config:
  stepca       (Traefik uses as certResolver — no HTTP routing needed)
  rss/miniflux (behind wg3 via VPN)
  vaultwarden  (behind wg3 via VPN)
  beets        (behind wg3 via VPN)
  cinny        (behind wg3 via VPN)
  navidrome    (behind wg3 via VPN)
  samba        (app-runc, no Traefik routing)


wg4_test      172.35.0.0/16  (TEMPORARY — VPN migration testing)
  (default project bridge)   wg4          wireguard (test tunnel)
               network_mode service:wg4  wg4-test (busybox, exec to check VPN)
